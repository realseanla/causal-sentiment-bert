Penetration Testing is a methodology for assessing network security, by generating and executing possible attacks.
Doing so automatically allows for regular and systematic testing.
A key question then is how to automatically generate the attacks.
A natural way to address this issue is as an attack planning problem.
In this thesis, we are concerned with the specific context of regular automated pentesting, and use the term "attack planning" in that sense.
